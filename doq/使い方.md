# 日報メーカー 使い方

## 開き方

以下のURLをブラウザで開きます。

```text
https://isao-yanagi.github.io/nippo-maker/
```

スマホ・PCどちらでも利用できます。

## 基本の流れ

1. 「対象日」に日付を入力します。
2. 基本開始、基本終了、休憩時間を確認します。
3. PJ名、常駐先企業名、出社場所、勤務場所、作業内容、所感を入力します。
4. 「7日分を作成」を押します。
5. 生成された各日の内容を必要に応じて編集します。
6. 「コピー：月〜金」または「コピー：月〜日」を押して、日報本文をコピーします。

## 各項目について

- 対象日: 入力した日付を含む週の月曜日から日曜日までを作成します。
- 基本開始 / 基本終了 / 休憩時間: 残業時間の自動計算に使います。
- 勤務場所: `(在宅)`、`(出社)`、`(全休)` から選びます。
- 項目表記の省略: `Yes` の場合は番号付きの簡易形式でコピーします。

## 全休を入力する場合

各日の「勤務場所」で `(全休)` を選ぶと、以下の項目が自動で `私用に付き全休` になります。

- PJ名
- 常駐先企業名
- 出社場所
- 作業内容
- 所感

`(全休)` から `(在宅)` または `(出社)` に戻すと、初期設定の値に戻ります。

## 残業時間について

残業時間は次の計算で自動表示されます。

```text
当日の勤務時間 - 基本勤務時間
```

勤務時間は、開始時刻・終了時刻・休憩時間から計算されます。

例:

```text
基本 09:00〜17:30、休憩 01:00
当日 09:00〜18:45、休憩 01:00
残業 1h15m
```

## 保存について

入力内容はブラウザの `localStorage` に自動保存されます。

保存キー:

```text
nippo-web-state-v11
```

保存先は利用者ごとの端末・ブラウザ内です。複数人が同じURLを開いても、通常は入力内容が互いに上書きされることはありません。

ただし、同じ端末・同じブラウザを複数人で使う場合は、同じ保存内容を共有する可能性があります。

## 初期設定のインポート / エクスポート

「初期設定をエクスポート」を押すと、現在の初期設定をJSONファイルとして保存できます。

「初期設定をインポート」を押すと、エクスポートしたJSONファイルを読み込み、初期設定欄へ反映できます。

インポート対象:

- 対象日
- 基本開始
- 基本終了
- 休憩時間
- PJ名
- 常駐先企業名
- 出社場所
- 勤務場所
- 作業内容
- 所感
- 項目表記の省略

## 初期値に戻す

画面の「初期値に戻す」を押すと、ブラウザに保存された入力内容を削除し、初期値に戻します。

## 注意点

- 別端末や別ブラウザには入力内容は引き継がれません。
- ブラウザのサイトデータを削除すると、保存内容も消えます。
- GitHub Pagesで公開しているため、アプリ本体はインターネット接続がある状態で開いてください。
